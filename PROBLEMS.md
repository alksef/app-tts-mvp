# App TTS - Расширенное описание проблемы

## Краткое резюме

Приложение для людей, использующих синтез речи (TTS) для общения, с минимальным отвлечением от работы за компьютером.

---

## Сценарии использования

- Общение в мессенджерах (Telegram, WhatsApp, Discord и др.)
- Работа в браузере
- Любое приложение на Windows
- Работа внутри самого приложения TTS

---

## Проблематика

### Проблемы существующих TTS решений

| Проблема | Описание | Последствия для пользователя |
|----------|----------|------------------------------|
| **Низкая скорость ввода** | Отсутствие механик для ускорения набора текста | Медленное общение, frustration |
| **Отсутствие интеграции** | TTS приложения работают изолированно | Необходимость переключаться между окнами, потеря контекста |
| **Сложность использования** | Нет быстрых клавиш, глобального перехвата | Высокий cognitive load, неудобство |

### Ключевое отличие (Unique Value Proposition)

**Интеграция с workflow** — приложение должно работать незаметно в фоновом режиме и быть доступным мгновенно по горячей клавише в любом приложении.

---

## Функциональные требования

### Критичные для MVP

#### 1. Быстрый перехват ввода (Keyboard Intercept)

**Режимы работы:**
- **В окне приложения:** Стандартный текстовый ввод
- **Глобальный перехват:** По горячей клавише перехватывать клавиатуру в любом приложении

**Технические требования (Windows):**
- Использование WinAPI для low-level keyboard hook
- Глобальная горячая клавиша для включения/выключения перехвата
- Опционально: показ/скрытие окна по горячей клавише

#### 2. Потоковый TTS

**Поведение:**
- При вводе terminating символов (`.`, `!`, `?`) отправлять предложение на TTS
- Опциональный режим (можно включать/выключать)

### Второстепенные (post-MVP)

- AI автодополнение (Anthropic Claude API)
- Слои клавиатуры (виртуальные макросы)
- История фраз в сессии
- Отдельная физическая клавиатура

---

## Технические требования

### Платформа
- **Только Windows** (использование WinAPI для keyboard hook)

### TTS движок
- **Облачные TTS сервисы** (Azure, Google, Amazon и др.)
- Приоритет качеству естественности голоса

### AI интеграция
- **Anthropic Claude API** для автодополнения текста
- Контекст: ранее введённый текст + суммаризация
- Предложить несколько вариантов продолжения

### Технологический стек
- **Не определён** — требуется исследование

---

## Нефункциональные требования

### Хранение данных
- История фраз: **только в сессии** (не сохраняется между запусками)

### Производительность
- Минимальная задержка между вводом и озвучкой
- Быстрое переключение между режимами

### UX
- Управление полностью через горячие клавиши
- Минимальное визуальное отвлечение
